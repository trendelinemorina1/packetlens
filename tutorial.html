PacketLens

// Live Network Traffic Analyzer

A real-time CLI tool that captures, analyzes, and flags suspicious network activity. Built for penetration testers and security researchers.

root required python 3.8+ scapy · rich kali linux
▼ scroll to install ▼
// overview

What It Does

📡

Live Capture

Sniffs packets in real-time on any interface — eth0, wlan0, lo, or custom.

📊

Protocol Stats

Breaks down TCP, UDP, ICMP, ARP, DNS, and HTTP packet counts and rates.

🔥

Top Talkers

Identifies the most active IP pairs and most-hit ports in real time.

🚨

Alert Engine

Detects port scans (15+ ports from one IP) and ARP spoofing attacks live.

💾

Export

Save full capture sessions to JSON or CSV for offline analysis.

🎛️

BPF Filters

Use standard Wireshark-style filters to focus on specific traffic.

// step 01

Install Dependencies

⚠ ROOT REQUIRED Raw packet sniffing requires root. Always run with sudo.
1

Update Kali & install pip

Make sure your system is up to date first.

$ sudo apt update && sudo apt upgrade -y
$ sudo apt install python3-pip -y
2

Install Python packages

Two packages needed: scapy (packet engine) and rich (terminal UI).

$ sudo pip3 install scapy rich --break-system-packages

Or use the included requirements file:

$ sudo pip3 install -r requirements.txt --break-system-packages
3

Make executable (optional)

So you can run it directly without typing python3 every time.

$ chmod +x packetlens.py
$ sudo ./packetlens.py # run directly
// step 02

Basic Usage

1

Find your network interface

You need to know which interface to listen on before starting.

$ ip a              # list all interfaces
$ iwconfig          # wireless interfaces only
// common interfaces eth0 → wired Ethernet  |  wlan0 → Wi-Fi  |  lo → loopback  |  tun0 → VPN
2

Run with live dashboard (default)

Auto-detects the default interface and shows the full live UI.

$ sudo python3 packetlens.py
3

Specify an interface

$ sudo python3 packetlens.py -i eth0
$ sudo python3 packetlens.py -i wlan0
4

Stop the capture

Press Ctrl+C — PacketLens will print a session summary before exiting.

// step 03

All Flags & Options

Flag Description Example
-i / --iface Network interface to sniff on -i eth0
-f / --filter BPF packet filter expression -f "tcp port 80"
-c / --count Stop after N packets (0 = unlimited) -c 500
-t / --timeout Stop after N seconds (0 = unlimited) -t 60
--export Export format: json or csv --export json
--out Output filename (no extension) --out my_capture
--no-ui Disable live dashboard (plain output) --no-ui
// step 04

Real-World Examples

Monitor HTTP traffic only

$ sudo python3 packetlens.py -i eth0 -f "tcp port 80"

Watch DNS queries for 30 seconds

$ sudo python3 packetlens.py -f "udp port 53" -t 30

Capture 1000 packets and export to JSON

$ sudo python3 packetlens.py -c 1000 --export json --out recon_session

Monitor traffic from a specific host

$ sudo python3 packetlens.py -f "host 192.168.1.50"

ARP monitoring (detect spoofing on LAN)

$ sudo python3 packetlens.py -f "arp" -i eth0

Headless mode (pipe or log to file)

$ sudo python3 packetlens.py --no-ui -t 120 --export csv
// step 05

Understanding Alerts

Port Scan Detection

PacketLens tracks destination ports per source IP. If a single host hits more than 15 unique ports, it raises an alert:

⚠  [PORT SCAN] 192.168.1.42 scanned 23 ports

ARP Spoof Detection

PacketLens builds an ARP table. If an IP's MAC address changes unexpectedly in a reply, it raises:

⚠  [ARP SPOOF] IP 192.168.1.1 changed MAC: aa:bb:cc → 11:22:33
// tip ARP spoof alerts are a strong indicator of a man-in-the-middle attack on your local network.
